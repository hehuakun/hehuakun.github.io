1 排序 （1） 最经典的要算冒泡排序了，冒泡排序的核心在于一个简单的思想：通过相邻位置的两两比较，总能找到待排序数列中的极值（极大值或极小值）。那么我们只要对剩下的所有待排序数列一直按这种两两比较的方法找到极值（注意是极值，不是最值），直到比较完最后两个数（即最终待比较数列只剩下一个数），此时就将待排序数列按照非增长或非减少顺序排列了。 还不清楚？下面再详解： 考虑最简单的情况，两个不同的数怎么找出其中较大的数？？ 当然是比较他们的大小，比较大的数就是较大的数。 然后考虑三个不同的数，怎么找出他们较大的数（注意只是找出较大的数，不是排序）？？ 假设三个数是A ，B， C，先比较A，B找出其中较大的数，然后将较大的数放在后面，再用这个较大的数去和C比较，然后就可以得到三个数中较大的数了。是不是？？ 然后依次类推，最后我们得出找到n个数中的极值需要比较（n-1）次 对这个算法进行分析，假设最坏的情况是待比较数列是按非增长的顺序排列的，现在我们要用冒泡排序按照非减少的顺序排列，那么我们在找这n个数中极大值时就需要比较（n-1）次，然后找剩下的（n-1）个待比较数列的极大值时需要比较（n-2）次，依次类推，直到最后两个数比较一次。算算我们比较了多少次 $ 1+2+3+...+(n-1)=n(n-1)/2 $次 这是该算法最坏情况的时间复杂度。 其实对于冒泡算法除了在比较完最后两个数结束外，还有一个结束条件，就是在一轮比较中，如果未发生交换，那么说明接下来的数已经是按照要求排列好的了。 比如，数列本身就是按照非增长排序的，现在仍然要求按照非增长排序。那么在找第一个极小值时，数列未发生交换，这个时候后面就不用比较了。 此时比较了 $n-1$次 下面上一段风骚细腻的代码 ~~~~ //此段代码摘自严蔚敏《数据结构》，我自己加上了注解 for(int i=n-1,bool change=true;i>0&change;--i)//外层循环有两个结束条件，达到循环（n-1）次数或者某次内层循环未交换。 { change=false;//在内层循环开始之前，将交换标志置为false，在确实发生交换后再置为true for(int j=1;j<=i;++j)//内层循环比较i次，这就是为什么外层循环写成从（n-1）到1递减的形式，而不是从1到（n-1）递增的形式 { if(A[j]>A[j+1]) A[j]<—>A[j+1];//前面比后面打就交换，说明是按照递增排序的 change=true;//发生了交换，就将交换标志位置为true } } ~~~~ （2）插入排序。想象在玩扑克牌的时候，在抓牌过程中，你是怎么让自己手上的牌保持一个非增长或非减少的顺序的？？ 你是抓到一张新牌，将之与手上的已经排好序的牌比较，找出新牌应该插入的位置插入新牌。这就是插入排序的思想。 下面仍是上代码 ~~~~ //此段代码来源于《算法导论》，我作了修改，使之符合c语言规范 //输入为数组A[0，n-1] ~~~~
